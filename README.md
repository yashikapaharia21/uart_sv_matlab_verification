# UART Transmitter — SystemVerilog + MATLAB Verification

A beginner-level hardware verification project implementing and testing a **UART Transmitter** in SystemVerilog, with MATLAB used to visualise the expected bit-stream waveform.

This was done as a learning exercise to understand how serial communication protocols work at the hardware level — covering FSM design, basic testbench writing, and connecting simulation to a MATLAB plot.

---

## What is UART?

UART (Universal Asynchronous Receiver-Transmitter) is one of the most common ways microcontrollers and computers send serial data. The 8N1 format used here means:

- **8** data bits per frame
- **N** — no parity bit
- **1** stop bit

```
 IDLE  START  D0   D1   D2   D3   D4   D5   D6   D7  STOP  IDLE
  1  |  0  |  ?  |  ?  |  ?  |  ?  |  ?  |  ?  |  ?  |  ?  |  1  |  1
       <--- bits sent LSB first ------------------------------------->
```

Each bit lasts one *bit period* = 1 / baud_rate. At 115200 baud that is about **8.68 µs**.

## How It Works

### RTL (`uart_tx.sv`)

A 4-state FSM that steps through each part of the UART frame:

| State | What Happens |
|---|---|
| `IDLE` | Line held HIGH, waiting for `tx_start` |
| `START` | Pulls line LOW for one bit period (start bit) |
| `DATA` | Outputs each data bit (LSB first) for one bit period each |
| `STOP` | Pulls line HIGH for one bit period, then asserts `tx_done` |

Baud rate is set by the `CLKS_PER_BIT` parameter:
- 25 MHz clock ÷ 115200 baud = **217 clock cycles per bit**

### Testbench (`uart_tx_tb.sv`)

Sends several known bytes and samples `tx_serial` at the mid-point of each bit period to check what actually came out. Compares against what was sent and reports PASS or FAIL.

Test cases:
- `0x41` — ASCII `'A'`
- `0x00` — all zeros
- `0xFF` — all ones
- `0xAA` — alternating `10101010`
- `0x55` — alternating `01010101`
- Two bytes back-to-back (`'B'` then `'C'`)

### MATLAB (`uart_analysis.m`)

Generates an annotated waveform plot showing exactly what the TX line should look like for a given byte — before running the simulation. This made it easy to cross-check the GTKWave output visually.

Also writes `sim/test_vectors.txt` as a plain-text reference of the tested byte values.

## How to Run

**Step 1 — Run MATLAB script** (generates waveform plot)
```matlab
cd matlab
uart_analysis
```

**Step 2 — Compile and simulate** (requires Icarus Verilog — free)
```bash
bash sim/run_sim.sh iverilog
```

**Step 3 — View waveform** (optional)
```bash
gtkwave sim/uart_tx_waves.vcd
```

Pre-generated results are in the `results/` folder if you just want to see the output.

## What I Learned

- How UART framing works at the bit level (start/stop bits, LSB-first ordering)
- Writing a basic FSM in SystemVerilog for timed serial output
- Sampling signals at mid-bit to reliably read logic levels (avoids edge glitches)
- Using MATLAB to model and visualise a digital signal before simulating it
